# Turkey Evaluation Output Spec

## Goal

Define one stable output contract for Turkish-mode evaluations so reports stay:

- practical for the candidate
- conservative when evidence is weak
- compatible with the existing evaluation pipeline spirit
- safer for later parser and dashboard work

## Scope

This spec applies to reports generated by `modes/tr/teklif.md`.

It does not change the underlying A-F evaluation structure.

## Output Principles

1. Do not overclaim.
2. If evidence is weak, say so.
3. Final recommendation must protect both candidate time and recruiter time.
4. The weighted score is not enough on its own; red-flag caps and confidence must be visible.

## Required Report Header

To reduce downstream breakage risk, keep these machine-stable keys in the report header even in Turkish mode:

```markdown
# Degerlendirme: {Company} -- {Role}

**Date:** {YYYY-MM-DD}
**Arquetipo:** {detected archetype}
**Score:** {final score}/5
**URL:** {original posting URL}
**PDF:** {pdf path or pending}
```

Notes:

- `**URL:**` is required.
- `**Arquetipo:**` stays fixed intentionally for compatibility with current parser assumptions.
- `**Score:**` should be the final score after any red-flag cap, not the raw weighted score.

## Required Body Structure

The body should keep the existing A-F sequence:

```markdown
## A) Rol Ozeti
## B) CV Eslesmesi
## C) Seniority ve Strateji
## D) Maas ve Piyasa
## E) Kisisellestirme Plani
## F) Mulakat Plani
```

After A-F, these sections are required:

```markdown
## Global Score
## Strengths
## Risks
## Recommendation
```

## Global Score Section

Required table:

```markdown
| Dimension | Weight | Score |
|---|---:|---:|
| Role Fit | 18 | X |
| Alignment With Candidate Goals | 12 | X |
| Seniority Fit | 10 | X |
| City / Work Model Fit | 10 | X |
| Language Fit | 8 | X |
| Salary Transparency / Market Fairness | 12 | X |
| Posting Quality | 8 | X |
| Company Clarity / Hiring Credibility | 8 | X |
| Application Effort | 6 | X |
| Interview Likelihood | 8 | X |
| **Weighted Score** | **100** | **X.XX/5** |
```

Required metadata directly below the table:

```markdown
**Red Flag Cap:** none | major | critical
**Final Score:** X.XX/5
**Confidence:** high | medium | low
**Recommendation Category:** hemen_basvur | secici_basvur | sinirda_once_dogrula | basvurma
**Borderline:** yes | no
```

Rules:

- `Final Score` is the action-driving score.
- If a cap exists, `Final Score` must reflect the capped result.
- `Borderline: yes` is mandatory when the case is not cleanly actionable.

## Strengths Section

Purpose:

- show why the role may still be worth attention

Rules:

- 3 to 5 bullets
- every bullet must be evidence-based
- prefer concrete fit signals over generic praise
- do not pad with weak positives

Good example:

- Direct match on production backend ownership, payment reliability, and incident response.

Bad example:

- Strong company and exciting opportunity.

## Risks Section

Purpose:

- make the decision honest

Rules:

- 3 to 5 bullets
- unknowns must be labeled as unknowns
- major or critical risks must be explicit
- if a risk is mitigable, note the mitigation briefly

Good example:

- Salary is fully opaque; available market data suggests the likely range may fall below target.

Bad example:

- Salary might be fine but hard to know.

## Recommendation Section

Required content:

1. One-sentence decision.
2. One short rationale.
3. If needed, 1-3 verification steps before applying.

Recommended shape:

```markdown
## Recommendation

Secici basvur. Rol mantikli, ama maas seffafligi ve hibrit beklentisi netlestirilmeden agresif ilerleme onerilmez.

Once dogrula:
- Hibrit gun beklentisi
- Maas bandi ve gross/net yapisi
- Rolun gercek raporlama hatti
```

## Conservative Logic

### Borderline

A report must set `Borderline: yes` if any of these is true:

- `Final Score` is between `3.7` and `4.1`
- confidence is `low`
- at least one major red flag exists
- two or more dimensions score `3/5`

Effect:

- recommendation category cannot be `hemen_basvur`

### Low Confidence

Use `Confidence: low` when:

- JD is incomplete or low-quality
- company identity is unclear
- location/work model is unclear
- compensation evidence is weak
- multiple sources contradict each other
- 3 or more dimensions were scored with weak evidence

Effect:

- recommendation category cannot be `hemen_basvur`
- recommendation should explicitly say what is missing
- tone must stay cautious

### Recommendation Category Thresholds

Use these thresholds:

- `hemen_basvur`: final score `>= 4.5`, no major/critical red flag, confidence not low
- `secici_basvur`: final score `4.0-4.49`, no critical red flag
- `sinirda_once_dogrula`: final score `3.5-3.99`, or any borderline case, or any low-confidence case with score `>= 4.0`
- `basvurma`: final score `< 3.5`, or any critical red flag

If thresholds and caution rules conflict, the more conservative outcome wins.

## Draft Application Answers Gate

The Turkish mode should only produce a draft-answers section when all are true:

- `Final Score >= 4.5`
- `Confidence` is not `low`
- no major or critical red flag exists

Otherwise, the report should stop at the recommendation and avoid overcommitting the candidate.
